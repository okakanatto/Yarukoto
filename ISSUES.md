## ✅ 解決済みのブロッカー

### 起動時の Database Locked エラー (Race Condition)
**状態**: ✅ 解決済み (2026-02-23)
**修正内容**:
- `lib/holidayService.js`: `updateHolidayCache` が500回の `INSERT` を個別に発行し、TauriのRust側 SQLite コネクションプールを枯渇（デッドロック）させていた構造的な問題を特定。
- 解決策として、対象となる約500日分の祝日データを動的に連結させた **単一のBulk（バルク）INSERT処理** へと書き換え、通信・ロックの保持時間を劇的に短縮して完全解消。
- さらに、アプリ起動ごとの再フェッチを防ぐため、`app_settings` の `holiday_last_fetch_date` を用いた **30日間の自動キャッシュ更新機構** を実装。

## 🐛 未解決の不具合 (Bugs)

### BUG-1. 「＋」ボタンからの子タスク追加時にタグが継承されない
- **画面**: タスク一覧
- **状態**: 🔴 未修正
- **報告日**: 2026-02-24
- **内容**: `app_settings` の「子タスクに親タスクのタグを自動付与」(`inherit_parent_tags`) がONの場合でも、タスク一覧で親タスク横の「＋」ボタンからインラインで子タスクを追加した際に、親タスクのタグが子タスクに付与されない。
- **期待動作**: オプションがONであれば、DnDによるネスト時と同様に、この追加経路でも親タスクのタグが自動的に子タスクに付与されるべき。
- **関連ファイル**: `components/TaskInput.js`, `components/TaskList.js`

### BUG-2. データ全削除時にルーティンが削除されない
- **画面**: 設定
- **状態**: 🔴 未修正（Tauri移行時の退行バグ）
- **報告日**: 2026-02-24
- **内容**: 設定画面の「データ全削除」を実行しても、ルーティンデータ（`routines`, `routine_tags`, `routine_completions`）が削除されない。v1.0以前のElectron/API Routes時代に修正済みだったが、Tauri IPC移行時に修正が抜け落ちた模様。
- **期待動作**: データ全削除はルーティン関連テーブルも含めてすべてのユーザーデータを削除する。
- **関連ファイル**: `app/settings/page.js`

## 🔧 機能改善 (Improvements)

### IMP-1. 「やるタスク」画面からのタスク編集
- **画面**: やるタスク
- **優先度**: 高
- **内容**: 「やるタスク」画面から直接タスクの内容を編集できるようにする（削除機能は不要）。
- **ユースケース**: 備考欄へのメモ追記、終わらないタスクの終了日変更など、日常的な微修正をタスク一覧に遷移せず行いたい。
- **実装方針**: 既存の `TaskEditModal` を「やるタスク」画面でも呼び出せるようにする。

### IMP-2. アーカイブ機能の実装
- **画面**: タスク一覧
- **優先度**: 高
- **内容**: 完了・キャンセル済みタスクが大量に溜まるのを防ぐため、フィルタリングとは別にアーカイブ機能を実装する。
- **仕様詳細**:
  - **手動アーカイブ**: 完了またはキャンセル済みのタスクに対して、アーカイブボタンを表示する（削除ボタンの横に配置するイメージ）。未完了タスクはアーカイブ不可。
  - **自動アーカイブ（オプション）**: 「完了日（`completed_at`）からN日経過したらアーカイブ」をオプション設定で有効化できる。設定変更時に既存の対象タスクも遡ってアーカイブされる。完了即アーカイブはしない（タスク消化の実感が失われるため）。
  - **キャンセルタスク**: キャンセル済みタスクは `completed_at` を持たないため、自動アーカイブの対象外。手動アーカイブのみ可能。
  - **親子タスクの制御**: 親タスクは子タスクがすべて完了またはキャンセルになっていないとアーカイブ不可。親タスクをアーカイブすると、子タスクもまとめてアーカイブされる。
  - **閲覧・復元**: アーカイブ済みタスクは「アーカイブ済み」タブから一覧表示でき、復元（アーカイブ解除）も可能にする。
- **DB設計案**: `tasks` テーブルに `archived_at` (DATETIME, NULL許容) カラムを追加。NULLでないものがアーカイブ済み。`app_settings` に自動アーカイブ日数を保存。
- **関連**: `completed_at` の各画面への表示も必要（IMP-11参照）。

### IMP-3. フィルター条件の複数選択
- **画面**: やるタスク, タスク一覧
- **優先度**: 高
- **内容**: フィルター条件を複数選択できるようにする。現在は各フィルタが単一選択のみだが、特にステータスにおいて「完了」と「キャンセル」を除外して表示する、といった複数条件のフィルタリングを可能にしたい。
- **対象フィルタ**: ステータス、タグ、重要度、緊急度すべて。

### IMP-4. ソートのON/OFF切替と手動並び替え
- **画面**: やるタスク, タスク一覧
- **優先度**: 中
- **内容**: 自動ソート（ステータス順・タグ順等）のON/OFFを切り替えられるようにし、OFFの場合はドラッグ&ドロップによる手動並び替えを可能にする。
- **仕様詳細**:
  - 並び替えは「親タスク単位」「子タスク単位」それぞれ独立して行える。
  - 「やるタスク」「タスク一覧」それぞれで独立した並び順を保持する。
  - 自動ソートをONにした場合、手動の並び順は解除されて構わない。
  - 新規タスク追加時はリストの**先頭**に挿入される。
  - （将来的には手動の並び順を裏で保持し、自動ソートOFF時に復元する拡張も検討）。
- **DB設計案**: `tasks` テーブルに `sort_order` カラムを追加、または別テーブルで画面別の並び順を管理。

### IMP-5. タスク入力画面のUI改善（項目の優先順位整理）
- **画面**: タスク入力（TaskInput）
- **優先度**: 中
- **内容**: 最低限入力が必要な項目と拡張的な項目を視覚的に分離し、入力のハードルを下げる。
- **項目の表示順**:
  1. タスク名（必須）
  2. 親タスク（該当時のみ）
  3. 終了期限
  4. 備考
  5. タグ
  6. 開始日、想定工数
  7. 重要度、緊急度
- **実装方針**: 上位項目は常時表示、下位項目は「詳細設定」等のアコーディオンで折りたたむ形を想定。

### IMP-6. 「期限切れタスクを今日やるタスクに表示する」オプション
- **画面**: 設定, やるタスク
- **優先度**: 中
- **内容**: 現状では `due_date` が当日以前の未完了タスクがすべて「やるタスク」に表示されるが、当日分のみ表示するオプションを追加する。
- **仕様詳細**:
  - `app_settings` に `show_overdue_in_today` キーを追加（デフォルト: `'1'` = 表示する = 現状維持）。
  - `'0'` に設定すると、`due_date` が厳密に当日と一致するタスクのみ自動表示される。
  - ON/OFFの切り替えはデータ構造に影響を与えない（表示フィルタのみの変更）。

### IMP-7. タグのアーカイブ（非表示化）
- **画面**: 設定, タスク入力
- **優先度**: 低
- **内容**: 不要になったタグを、完了済みタスクのデータに影響を与えずに各種一覧（設定画面のタグ管理・タスク入力画面のタグ選択ドロップダウン）から非表示にできるようにする。
- **DB設計案**: `tags` テーブルに `archived` (INTEGER, DEFAULT 0) カラムを追加。アーカイブ済みタグは選択肢に表示されないが、既存の `task_tags` レコードはそのまま保持。

### IMP-8. テーマカラーの変更
- **画面**: 設定, 全体
- **優先度**: 低
- **内容**: ライトモード / ダークモード に加えて、モダンなデザインの配色パターンを5種類程度用意し、ユーザーが好みのテーマを選択できるようにする。
- **実装方針**: `globals.css` のCSS変数をテーマごとに切り替える。`app_settings` にテーマ設定を保存。

### IMP-9. 操作時のトースト通知
- **画面**: 全般
- **優先度**: 低
- **内容**: タスクに対する主要操作の完了時に、画面下部にトーストメッセージを約3秒間表示する。
- **対象アクション**:
  - タスク追加:「タスクを追加しました」
  - タスク編集:「タスクを更新しました」
  - タスク削除:「タスクを削除しました」
  - タスクアーカイブ:「タスクをアーカイブしました」
- **実装方針**: 軽量なトーストコンポーネントを自前で実装（ライブラリ不使用を想定）。FABからの追加、タスク一覧からの追加など全経路で統一的に表示。

### IMP-10. キャンセルステータスの追加
- **画面**: 全般
- **優先度**: 中
- **内容**: 既存の3ステート（未着手/着手中/完了）に加え、「キャンセル」ステータスを追加する。
- **仕様詳細**:
  - `status_master` に status_code = 4 として「キャンセル」を追加。システム必須ステータスとして削除・名前変更不可（1〜3と同様）。
  - **見た目**: 完了と同様にタスク名に取り消し線が引かれ、視覚的に「終了した」状態であることが分かる外観にする。
  - **操作方法**: 編集モーダルやタスクカードのステータスドロップダウンからのみ変更可能。3ステートチェックボックスのUIは変更しない。
  - **チェックボックスの挙動**: キャンセル状態のタスクではチェックボックスはグレーアウトして操作不可にする（ドロップダウンでのみステータス変更可能）。
  - **「やるタスク」画面**: キャンセル済みタスクは表示しない（完了タスクはやったことの可視化のため表示するが、キャンセルは表示対象外）。
  - **ダッシュボード集計**: キャンセルタスクは完了率の分母・分子いずれにも含まない（集計対象外）。

### IMP-11. 完了日（completed_at）の各画面への表示
- **画面**: タスク一覧, やるタスク, タスク編集モーダル
- **優先度**: 高（IMP-2 アーカイブ機能の前提）
- **内容**: タスクの実際の完了日を各画面で視認できるようにする。
- **仕様詳細**:
  - `completed_at` はタスクが完了ステータス(3)になった時点で自動的にセットされる（既存動作）。
  - 完了ステータスが解除された場合は自動的にNULLに戻る。
  - ユーザーによる手動での日付変更は不可（システム自動管理のみ）。
  - タスクカードおよび編集モーダルに、完了済みタスクの場合は完了日を読み取り専用で表示する。

### ENH-1. ダッシュボード: 今日完了したタスクの可視化
- **画面**: ダッシュボード
- **内容**: 今日予定のタスク（終了日が今日＋今日やる）だけでなく、今日実際に完了したタスクがあれば分かるようにしたい。
- **備考**: 具体的なUIイメージは未定。実装時に提案ベースで検討。

### ENH-2. プロジェクトによるフォルダ分け
- **画面**: タスク一覧, ルーティン, やるタスク
- **内容**: タスクおよびルーティンを「プロジェクト」単位でグルーピングする。
- **仕様詳細**:
  - **階層構造**: 「タスク一覧」画面 → プロジェクト → 親タスク → 子タスク → 孫タスク。「ルーティン」画面 → プロジェクト → ルーティンタスク。
  - **デフォルト**: 初期状態では「General」プロジェクトのみ存在。ユーザーが必要に応じてプロジェクトを追加できる。
  - **プロジェクトのアーカイブ**: プロジェクト単位でアーカイブ可能。アーカイブしたプロジェクトは別途参照できる。
  - **やるタスク画面での表示**: 子タスクのみ・孫タスクのみの場合でも、親タスク名やプロジェクト名が分かるような表示にする。
- **DB設計案**: `projects` テーブル新設 (id, name, archived, sort_order, created_at)。`tasks` と `routines` テーブルに `project_id` カラムを追加（デフォルトは General プロジェクトのID）。

### ENH-3. ユーザー定義カスタムフィールド
- **画面**: 設定, タスク入力, タスク一覧
- **内容**: タスクの入力項目をユーザーが設定画面から自由に追加・削除できるようにする。
- **対応フィールドタイプ**:
  - テキスト項目（自由入力）
  - リスト項目（設定画面で選択肢を定義）
  - 数字項目（数値入力）
- **DB設計案**: `custom_fields` テーブル (id, name, field_type, options_json, sort_order) と `task_custom_values` テーブル (task_id, field_id, value) でEAV構造を想定。

### ENH-4. 「やったタスク」画面（新規画面）
- **画面**: 新規追加
- **内容**: タスクの実際の完了日（`completed_at`）をもとに、過去の「やったこと」をカレンダービューで振り返れる画面を新設する。
- **備考**: サイドバーナビに新しい項目として追加。日別・週別・月別のビュー切り替えなど、具体的なUIは実装時に検討。

### ENH-5. 「子タスク全完了で親タスクも完了にする」オプション
- **画面**: 設定
- **内容**: `app_settings` に新オプションを追加し、ONの場合は子タスクがすべて完了状態になった時点で親タスクのステータスも自動的に完了に変更する。

## 📝 検討中・保留中の機能 (Backlog / Ideas)

### 第三階層までのタスク分解（孫タスク対応）
- **内容**: 現在の1階層（親→子）に加え、もう1階層（親→子→孫）のタスク分解を可能にする。
- **仕様メモ**:
  - 子タスクは孫タスクを持てるようになる。
  - 孫タスクはその下にタスクを持てない（現在の子タスクと同じ制限）。
  - `tasks` テーブルの `parent_id` 構造はそのまま活用可能（孫の `parent_id` → 子の `id`）。ただし3階層目以降の作成をUIおよびバリデーションで制限する。

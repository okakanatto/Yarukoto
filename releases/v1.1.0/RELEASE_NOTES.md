# Yarukoto v1.1.0 リリースノート

Yarukoto v1.1.0 をリリースしました！🎉
本バージョンでは、皆様からいただいた要望をもとに、操作性を向上させる多数の細かな機能改善と、安定性向上のための重要なバグ修正を行っています。

## 🌟 新機能・改善点 (Improvements)

* **[IMP-1] タスクの直接編集**: 「今日やるタスク」画面で、タスク名（完了チェックボックスの右側）をクリックすることで、直接編集モーダルが開けるようになりました。わざわざ「タスク一覧」画面に戻る必要がなくなりました。
* **[IMP-5] タスク入力UIの最適化**: タスク入力時の項目順序を「親タスク設定→終了期限→タグ→開始日・工数→重要度・緊急度→備考」という、現実の思考フローに近い自然な順番に整理しました。
* **[IMP-6] 期限切れタスクの表示オプション**: 過去の日付に設定されたまま未完了になっているタスクを「今日やるタスク」画面に引き継いで表示するかどうかを、「設定」画面のトグルスイッチで変更できるようになりました。
* **[IMP-9] グローバルトースト通知**: タスクの追加・編集・削除などを行った際に、画面右下（追加ボタンの右上）に成功・失敗を伝えるポップアップ通知（トースト）が表示されるようになり、操作のフィードバックが分かりやすくなりました。
* **[IMP-10] キャンセルステータスの導入**: タスクのステータスに「キャンセル」を追加しました。完了(Done)とは明確に区別され、UI上ではグレーアウト＋取り消し線で表示されます。ダッシュボードの進捗率(%）の計算母数からも除外されます。
* **[IMP-11] 完了日の可視化**: タスクを完了状態にした日（`completed_at`）がバックグラウンドで記録されるようになり、タスク一覧や「今日やるタスク」画面のカード内、および編集モーダルで「☑ 完了: YYYY-MM-DD」として確認できるようになりました。

## 🛠 バグ修正 (Bug Fixes)

* **[BUG-1] 子タスクのタグ継承漏れ**: タスク一覧の「＋」ボタンから特定タスクの子タスクを直接追加した場合に、設定（オプション）の「親タスクのタグを継承する」がONになっていてもタグが引き継がれない不具合を修正しました。
* **[BUG-2] 全データ削除不備**: 設定画面から「すべてのデータを削除」を実行した際に、ルーティンの設定データだけが消えずに残ってしまう不具合を修正しました。
* **[BUG-V1] 設定・名称変更の競合**: 設定画面でタグやステータスの名前を編集する際、キーボード入力のたびにデータベースへの保存が走り、入力がカクついたりデータが競合を起こす問題を修正しました（フォーカスが外れたタイミングで一括保存されます）。
* **[BUG-V2] 二重送信の防止**: タスクやルーティンを追加する際、Enterキーを極めて高速で連打すると、同じタスクが2〜3個同時に追加されてしまう問題を修正しました。
* **[BUG-V3] 致命的エラー時の白画面停止**: 初回起動時などで何らかの理由によりデータベースファイル(`tasks.db`)へのアクセス権限がなくエラーになった際、「ローディング中...」のままアプリがフリーズする問題を修正し、専用のフォールバックエラー画面を表示するようにしました。

## 💾 ダウンロードとアップデート方法

* **以前のバージョン（v1.0.0等）をお使いの方へ**
  * 今回のアップデートでは、データベースの互換性は完全に保たれています。ポータブル版・インストーラー版のいずれで移行しても、**以前のデータはそのまま自動で引き継がれ、安全に使い続けることができます**。

* **インストーラー版 (Yarukoto_1.1.0_x64-setup.exe)**
  * はじめてお使いの方や、スタートメニューへの登録・アンインストールをOSのコントロールパネルから管理したい方向けです。

* **ポータブル版 (Yarukoto_1.1.0-portable.exe)**
  * インストール不要で、ダウンロードしたEXEファイルをダブルクリックするだけで直ぐに起動します。USBメモリに入れて持ち運ぶ用途などに向いています。

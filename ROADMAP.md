# ROADMAP.md - Yarukoto 開発ロードマップ

v1.0.0リリース後の課題・機能追加を段階的に実装するための計画。
各課題の詳細は `ISSUES.md` を参照。

## 方針

- **開発フロー**: バージョンの全内容を実装 → 検証 → リリース → 次バージョンの全内容を実装 → …を繰り返す
- **リリース単位**: 1セクション分（v1.2.0なら2-1〜2-4すべて）の実装・検証が完了した時点でリリース
- **枝番単位**: 1回のプロンプト指示でまとめて実装する範囲
- **DB互換性**: 全バージョンでALTER TABLE ADD / CREATE TABLE / INSERTのみ使用し、既存データとの後方互換を維持
- **「検討中・保留中」の3件**（孫タスク、フィードバック、開発者支援）は本ロードマップに含まない

## 検証ステップ定義

> 各STEPの詳細なプロンプトは `dev-flow-guide.md` のフェーズ4を参照。

| STEP | 内容 |
|------|------|
| STEP A | 機能検証（正常系 + 異常系・境界値 + 状態遷移・データ件数を一括検証） |
| STEP B | 品質レビュー（エラーハンドリング + UI/文言の一貫性を一括レビュー） |
| STEP C | 設計レビュー（目的適合性 + 実用性・耐久性。メジャーバージョンアップ時のみ） |
| STEP R | リグレッションテスト（既存機能の破壊がないか、影響範囲を特定して再確認） |

## v1.4.0 — 新画面・構造拡張

> ダッシュボード改善、新画面追加、プロジェクト管理、カスタムフィールドなど機能面の拡張。v1.3.0で整備した並び替え・アーカイブ機能が前提。

### 4-1. ダッシュボード改善 + 子タスク自動完了オプション
- **ENH-1**: ダッシュボードで今日完了したタスクを可視化（UI提案ベースで実装時に検討）

- **DB変更**: `app_settings` に自動完了オプション INSERT
- **検証**: STEP A + STEP B + STEP R（DB変更あり）
- **作業量**: 普通

### 4-2. 「やったタスク」画面（新規）
- **ENH-4**: `completed_at` をもとに過去の完了タスクをカレンダービューで振り返る新画面
  - サイドバーナビに追加
- **検証**: STEP A + STEP B + STEP R（UI変更あり）
- **作業量**: 重い

### 4-3. プロジェクト機能
- **ENH-2**: タスク・ルーティンをプロジェクト単位でフォルダ管理
  - デフォルトは「General」プロジェクト

  - 「やるタスク」画面でプロジェクト名・親タスク名を表示
- **DB変更**: `projects` テーブル新設、`tasks` / `routines` に `project_id` カラム追加（既存データはGeneralに自動割当）
- **検証**: STEP A + STEP B + STEP R（DB変更あり）
- **作業量**: 重い

### 4-4. ユーザー定義カスタムフィールド
- **ENH-3**: タスクの入力項目をユーザーが自由に追加・削除
  - テキスト・リスト・数字の3タイプ
- **DB変更**: `custom_fields` / `task_custom_values` テーブル新設
- **検証**: STEP A + STEP B + STEP R（バージョン最終枝番）
- **作業量**: 重い

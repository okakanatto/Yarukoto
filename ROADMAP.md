# ROADMAP.md - Yarukoto 開発ロードマップ

v1.0.0リリース後の課題・機能追加を段階的に実装するための計画。
各課題の詳細は `ISSUES.md` を参照。

## 方針

- **開発フロー**: バージョンの全内容を実装 → 検証 → リリース → 次バージョンの全内容を実装 → …を繰り返す
- **リリース単位**: 1セクション分（v1.2.0なら2-1〜2-4すべて）の実装・検証が完了した時点でリリース
- **枝番単位**: 1回のプロンプト指示でまとめて実装する範囲
- **DB互換性**: 全バージョンでALTER TABLE ADD / CREATE TABLE / INSERTのみ使用し、既存データとの後方互換を維持
- **「検討中・保留中」の3件**（孫タスク、フィードバック、開発者支援）は本ロードマップに含まない

## 検証ステップ定義

> 各STEPの詳細なプロンプトは `dev-flow-guide.md` のフェーズ4を参照。

| STEP | 内容 |
|------|------|
| STEP A | 機能検証（正常系 + 異常系・境界値 + 状態遷移・データ件数を一括検証） |
| STEP B | 品質レビュー（エラーハンドリング + UI/文言の一貫性を一括レビュー） |
| STEP C | 設計レビュー（目的適合性 + 実用性・耐久性。メジャーバージョンアップ時のみ） |
| STEP R | リグレッションテスト（既存機能の破壊がないか、影響範囲を特定して再確認） |

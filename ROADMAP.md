# ROADMAP.md - Yarukoto 開発ロードマップ

v1.0.0リリース後の課題・機能追加を段階的に実装するための計画。
各課題の詳細は `ISSUES.md` を参照。

## 方針

- **開発フロー**: バージョンの全内容を実装 → 検証 → リリース → 次バージョンの全内容を実装 → …を繰り返す
- **リリース単位**: 1セクション分（v1.1.0なら1-1〜1-4すべて）の実装・検証が完了した時点でリリース
- **枝番単位**: 1回のプロンプト指示でまとめて実装する範囲
- **DB互換性**: 全バージョンでALTER TABLE ADD / CREATE TABLE / INSERTのみ使用し、既存データとの後方互換を維持
- **「検討中・保留中」の3件**（孫タスク、フィードバック、開発者支援）は本ロードマップに含まない

## v1.1.0 — バグ修正 + コア基盤整備

> ステータスモデルの変更・表示改善・ユーティリティなど、後続機能の土台になるものを先に固める。

- **BUG-1**: 「＋」ボタンからの子タスク追加時にタグが継承されない

### 1-2. キャンセルステータス + 完了日表示
- **IMP-10**: キャンセルステータスの追加（status_code=4、グレーアウト+取り消し線）
- **IMP-11**: 完了日（`completed_at`）の各画面への表示（読み取り専用）
- **DB変更**: `status_master` にキャンセル(code=4) INSERT

### 1-3. トースト通知 + タスク入力UI改善
- **IMP-9**: 操作時のトースト通知（追加・編集・削除・アーカイブ、FABの上に表示）
- **IMP-5**: タスク入力フォームの項目順整理・詳細設定のアコーディオン化

### 1-4. やるタスク画面の改善 + 設定オプション追加
- **IMP-1**: 「やるタスク」画面からのタスク編集（削除不可）
- **IMP-6**: 「期限切れタスクを今日やるタスクに表示する」オプション
- **DB変更**: `app_settings` に `show_overdue_in_today` INSERT

## v1.2.0 — フィルタ・アーカイブ・並び替え

> タスク一覧の使い勝手に直結する中〜大規模機能群。v1.1で整備したキャンセルステータス・トースト・完了日表示が前提。

### 2-1. フィルタ複数選択 + タグアーカイブ
- **IMP-3**: フィルタの複数選択（ステータスは「完了・キャンセルを除く」トグル、他はチップON/OFF方式）
- **IMP-7**: 不要タグのアーカイブ（既存データに影響なく選択肢から非表示）
- **DB変更**: `tags` テーブルに `archived INTEGER DEFAULT 0` カラム追加

### 2-2. アーカイブ機能
- **IMP-2**: アーカイブ機能の実装
  - 手動アーカイブ（完了・キャンセル済みのみ）
  - 自動アーカイブ（`completed_at` からN日経過）
  - 親子連動アーカイブ（親アーカイブ→子もまとめてアーカイブ）
  - アーカイブ済みタブでの閲覧・復元
- **DB変更**: `tasks` に `archived_at TEXT` カラム追加、`app_settings` に自動アーカイブ日数 INSERT

### 2-3. ソートON/OFF + 手動並び替え
- **IMP-4**: 自動ソートのON/OFF切替 + DnDによる手動並び替え
  - DnD操作体系: ギャップ→並び替え、タスク上→ネスト、UnnestGap→アンネスト（共存）
  - 「やるタスク」「タスク一覧」それぞれで独立した並び順
  - 新規タスクはリスト先頭に挿入
- **DB変更**: `tasks` に `sort_order INTEGER` カラム追加

## v2.0.0 — テーマ・新機能・大型構造変更

> 見た目の刷新と、アプリの構造を拡張する大型機能。大きな変更のため最後にまとめて実施。

### 3-1. テーマカラー変更
- **IMP-8**: ライト/ダーク + モダンな配色5パターン程度
  - `globals.css` のCSS変数をテーマごとに切り替え
  - `app_settings` にテーマ設定を保存

### 3-2. ダッシュボード改善 + 子タスク自動完了オプション
- **ENH-1**: ダッシュボードで今日完了したタスクを可視化（UI提案ベースで実装時に検討）
- **ENH-5**: 「子タスクが全完了したら親タスクも完了にする」オプション

### 3-3. 「やったタスク」画面（新規）
- **ENH-4**: `completed_at` をもとに過去の完了タスクをカレンダービューで振り返る新画面
  - サイドバーナビに追加

### 3-4. プロジェクト機能
- **ENH-2**: タスク・ルーティンをプロジェクト単位でフォルダ管理
  - デフォルトは「General」プロジェクト
  - プロジェクトのアーカイブ・復元
  - 「やるタスク」画面でプロジェクト名・親タスク名を表示
- **DB変更**: `projects` テーブル新設、`tasks` / `routines` に `project_id` カラム追加（既存データはGeneralに自動割当）

### 3-5. ユーザー定義カスタムフィールド
- **ENH-3**: タスクの入力項目をユーザーが自由に追加・削除
  - テキスト・リスト・数字の3タイプ
- **DB変更**: `custom_fields` / `task_custom_values` テーブル新設

## 最新の作業（2026-03-01）

- **フェーズ**: v1.4.0 動作確認バグ修正（今日やるタスク画面の重複表示 — 3回目修正）
- **対象バージョン**: v1.4.0
- **対象課題**: 「今日やるタスク」画面で同名のタスクとルーティンが重複表示される不具合の修正
- **ステータス**: ✅ 完了
- **やったこと**:
  - **根本原因の特定**: ルーティンとタスクの**クロスエンティティ重複**。同名の「A社向け提案書のレビュー」がルーティン（`id: routine_X_date`）とタスク（`id: 数値`）として別エンティティで表示されていた。`useTodayTasks.js` の重複排除は ID ベースの `Set` のみで、異なるエンティティ種別間の同名重複を検知できなかった。サイドバー（`layout.js`）もタスクとルーティンを別SQLで取得して単純合算するため同じ問題を持ち、**件数が一致しているように見えて実際は両方が水増し**されていた
  - **前回修正（weekdays_only チェック）の効果**: 件数を9→7に削減（weekdays_only=1 の daily ルーティン2件を正しく除外）。しかし weekdays_only=0 の同名ルーティンは引き続き表示され、根本の重複は未解消だった
  - **修正内容**:
    1. `hooks/useTodayTasks.js`: マージ前にタスクのタイトル Set を構築し、同名ルーティンをフィルタアウト（タスク側優先 — ステータス・期限日・親タスク等メタデータが豊富なため）
    2. `app/layout.js`: サイドバーのタスク SQL に `title` カラム追加、ルーティン SQL に `r.title` カラム追加。カウント時に同様のタイトルベースのクロスエンティティ重複排除を実施
  - `npm run lint` 実行、エラーなし
- **変更したファイル**: `hooks/useTodayTasks.js`, `app/layout.js`
- **影響範囲**: 「今日やるタスク」画面のタスク一覧表示とサイドバーの進捗カウント。タスク一覧・ダッシュボード・ルーティン管理画面は別のデータ取得ロジックを使用しているため影響なし
- **次にやるべきこと**:
  - 実機で修正を確認する（同名のタスクとルーティンが1件のみ表示されること、サイドバー件数とメインページ件数が一致すること）
  - フェーズ5（動作確認）を続行する
- **注意事項・申し送り**:
  - 本修正は**タイトル完全一致**で重複判定している。ユーザーが意図的に同名のタスクとルーティンを別物として作成した場合、ルーティン側が非表示になる。現実的にはほぼ発生しないケースだが、将来的にプロジェクト機能（v1.5.0 ENH-2）導入時に再検討の余地あり
  - 前回・前々回の修正（weekdays_only チェック追加、showOverdue 初期値修正、masterDataReady ガード、Set 重複排除）はすべてそのまま維持。それぞれ独立した正当性がある
  - **3回の修正の経緯**: ①useTodayTasks.js 防御的修正（原因分析誤り）→ ②holidayService.js weekdays_only 修正（部分解消、9→7件）→ ③クロスエンティティ重複排除（今回、根本解消）
